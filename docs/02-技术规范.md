# 技术规范 - Diary App

## 技术选型原则

- **简单优先**：用户是代码小白，方案越简单越好
- **无需构建**：不引入复杂构建工具，直接写原生代码
- **纯前端**：所有功能在浏览器完成，无需服务器

---

## 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 结构 | HTML5 | 标准 HTML 语义化标签 |
| 样式 | CSS3 | 原生 CSS，使用 CSS 变量管理主题色 |
| 逻辑 | Vanilla JavaScript | 原生 JS，不引入大型框架 |
| 存储 | IndexedDB | 浏览器本地数据库，支持大容量存储 |
| 图片 | Canvas API | 前端压缩图片，控制存储大小 |

---

## 数据存储方案

### IndexedDB 结构设计

```
database: diaryApp
  |
  ├── store: entries (日记表)
  |     ├── id: auto increment
  |     ├── title: string
  |     ├── content: string
  |     ├── images: array<string> (base64)
  |     ├── createdAt: timestamp
  |     ├── updatedAt: timestamp
  |     ├── isPinned: boolean
  |     └── dateKey: string (YYYY-MM-DD 格式，用于日历)
  |
  └── store: metadata (元数据)
        ├── lastBackup: timestamp
        └── entryCount: number
```

### 存储容量估算

- 纯文本：1 篇日记 ≈ 10KB
- 含 1 张图片：1 篇日记 ≈ 200KB
- IndexedDB 限制：约 50-60MB（各浏览器不同）
- 估算可存 200-300 篇含图日记

### 图片处理策略

1. **压缩**：使用 Canvas 将图片压缩至最大 1200px 宽度
2. **格式**：转换为 JPEG，质量 80%
3. **存储**：转为 base64 字符串存入 IndexedDB

### 文件结构

```
diary-app/
├── index.html          # 主入口文件
├── css/
│   └── style.css       # 主样式文件
├── js/
│   ├── app.js          # 应用主逻辑
│   ├── db.js           # IndexedDB 操作
│   ├── calendar.js     # 日历组件逻辑
│   └── utils.js        # 工具函数
├── assets/
│   └── images/         # 占位图片资源
└── ...
```

### 命名规范

- **文件**：小写，单词用连字符分隔，如 `date-utils.js`
- **变量**：驼峰命名，如 `entryList`, `isPinned`
- **常量**：全大写下划线，如 `MAX_IMAGE_SIZE`
- **函数**：动词开头，如 `saveEntry()`, `renderList()`
- **CSS 类**：BEM 命名，如 `.entry-card__title`

## 安全注意事项

1. **XSS 防护**：用户输入内容需要进行 HTML 转义
2. **数据验证**：导入时验证 JSON 格式，防止错误数据
3. **存储检查**：操作前检查 IndexedDB 是否可用
